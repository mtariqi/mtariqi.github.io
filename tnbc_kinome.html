About

This project portfolio addresses a core challenge in precision oncology: pathway redundancy in Triple-Negative Breast Cancer (TNBC). When one kinase is blocked by a targeted therapy, cancer cells frequently activate a redundant kinase feeding the same downstream pathway — the primary mechanism behind acquired resistance.

Two complementary tools are developed here. FRAS-90 provides a systems-level view of the entire 90-kinase tyrosine kinome using multi-omic redundancy scoring. TNBC Genomics Agent provides patient-level VCF-driven analysis with an AI reasoning layer for clinical narrative generation.

Both tools are open-source, locally deployable, and designed so patient genomic data never leaves the researcher's environment. The AI layer (Claude) acts as a reasoning engine over structured data, not a knowledge database — following the principle articulated by Truhn, Reis-Filho & Kather, Nature Medicine 2023.

Systems-level · All 90 kinases

FRAS-90 — Functional Redundancy
Analysis System

Multi-omic Rs scoring across the complete human tyrosine kinome. Integrates structural homology (UniProt/MUSCLE), transcriptomic co-expression (TCGA-TNBC), and phosphoproteomic substrate overlap (CPTAC) into a single weighted redundancy score.

+
Rs = (0.3 × S) + (0.3 × ρ) + (0.4 × J)
ComponentWeightMethodSource
S — Structural similarity30%MUSCLE MSA · catalytic domain sequence identityUniProt
ρ — Transcriptomic correlation30%Spearman rank · RNA-Seq across TNBC samplesTCGA-BRCA
J — Substrate Jaccard overlap40%|PhosA ∩ PhosB| / |PhosA ∪ PhosB|CPTAC

J receives the highest weight because shared phosphorylation substrates represent the direct mechanistic basis for bypass resistance. Structural similarity and transcriptomic co-regulation provide supporting evidence for predicting redundancy before resistance emerges.

1
Structural homology mapping
MUSCLE MSA of UniProt kinase catalytic domain sequences (~250–300 aa). Generates a 90×90 pairwise sequence identity matrix S.
python scripts/build_tree.py --input kinase_sequences.fasta --method muscle
2
Transcriptomic co-expression analysis
Spearman rank correlation of RNA-Seq (log₂ TPM) across TCGA-BRCA triple-negative cohort (ER−/PR−/HER2−, n>400). Generates ρ matrix.
python scripts/analyzer.py --mode transcriptomic --cohort TCGA-TNBC --metric spearman
3
Phosphoproteomic substrate convergence
Jaccard index from CPTAC BRCA phosphorylation data — motif matching + experimental validation. Generates J matrix. Highest weight component.
python scripts/analyzer.py --mode proteomic --dataset CPTAC-BRCA --metric jaccard
4
Rs synthesis & clinical reporting
Weighted integration of S, ρ, J matrices. Grade I–IV classification. TNBC combination therapy recommendations. AI narrative via Claude.
python pipeline.py --gene EGFR --out results/report.json

Patient-level · VCF-driven

TNBC Genomics Agent

End-to-end pipeline from patient VCF files to combination therapy recommendations. Parses variants, annotates against a curated 10-kinase database, scores pathway redundancy, and synthesizes findings via an AI reasoning layer.

+
1
VCF parsing
Parses VCF 4.x files. Extracts all variants, annotates against GENE + EFFECT INFO fields. Severity scoring from VEP-style consequence terms (stop_gained → 1.0, missense → 0.70).
2
Kinase annotation
Cross-references detected genes against curated 10-kinase database. Attaches TNBC frequency, pathway membership, inhibitor list, and known resistance mechanisms.
3
Redundancy scoring
Sigmoid-based pathway redundancy: score = 1 − (1 / (1 + 0.8×(n−1))). Per-gene bypass risk from co-affected kinase count. Combination therapy ranking by evidence strength.
4
AI clinical synthesis
Structured VCF summary + redundancy report sent to Claude (claude-sonnet-4-6). Six-section clinical narrative output. All facts grounded in pipeline data — no hallucination risk.
python pipeline.py --vcf patient_1.vcf --out reports/patient_1_report.json

The AI layer uses Claude (claude-sonnet-4-6) as a reasoning engine over structured bioinformatics output — not as a knowledge source. This eliminates hallucination risk for clinical data.

Design principle (Truhn, Reis-Filho & Kather, Nature Medicine 2023): LLMs should be used as scientific reasoning engines, not knowledge databases. All clinical facts originate from the curated kinase database and the patient's VCF. Claude's role is synthesis, prioritisation, and narrative generation only.
Prompt architecture:
System: "You are an expert TNBC bioinformatician. Reason only from the provided structured input."
User: JSON(vcf_summary) + JSON(redundancy_report)
Output: 6-section clinical narrative (variants → pathways → targets → combinations → resistance → priority action)
Privacy: All bioinformatics computation runs locally. Only the de-identified structured summary (gene names, pathway scores) is sent to the API. Real patient VCF files are gitignored — accidental exposure is structurally impossible.
